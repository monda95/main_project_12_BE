## popular_queries_mv (MV)
> SearchLog에서 집계된 인기 검색어.  
> Celery Beat 배치 작업(`update_popular_queries`)으로 주기적 업데이트.  
> Django 모델은 `managed = False`.

| 컬럼 | 타입 | NULL | 설명 |
|---|---|---:|---|
| query | TEXT | NO | 정규화된 검색어 |
| cnt | BIGINT | NO | 등장 횟수 |
| last_seen | TIMESTAMPTZ | NO | 마지막 등장 시각 |

인덱스: `(query UNIQUE)`

---

## recommended_questions_mv (MV)
> 특정 검색어(query)에 대한 추천 질문 목록.  
> Celery Beat 배치 작업에서 생성/갱신.  
> Django 모델은 `managed = False`.

| 컬럼 | 타입 | NULL | 설명 |
|---|---|---:|---|
| query | TEXT | NO | 기준 검색어 |
| suggestions | JSONB | NO | 추천 질문 배열 |
| created_at | TIMESTAMPTZ | NO | 생성 시각 |

인덱스: `(query, created_at DESC)`

## 사전집계 정책 (예시)
```sql
-- popular_queries_mv 정의 예시
-- 주의: 직접 INSERT/UPDATE 불가, Celery beat 배치에서 refresh 수행
CREATE MATERIALIZED VIEW popular_queries_mv AS
SELECT normalized_query AS query, COUNT(*)::bigint AS cnt, MAX(created_at) AS last_seen
FROM search_logs
GROUP BY normalized_query;

```

## 주요 인덱스
```sql
-- 이메일 대소문자 무시 고유 인덱스
CREATE UNIQUE INDEX ux_users_email_lower ON users ((LOWER(email)));

-- 성능 최적화 인덱스 
CREATE INDEX ix_conversations_owner_updated ON conversations (owner_id, updated_at);
CREATE INDEX ix_messages_conv_created ON messages (conversation_id, created_at);
CREATE INDEX ix_search_logs_created ON search_logs (created_at);

-- 사전집계 MV 인덱스
CREATE UNIQUE INDEX ux_popular_queries_mv_query ON popular_queries_mv (query);

```

docker compose exec web uv run python manage.py update_popular_queries 를 배포 시 celery beat로 자동화 계획
