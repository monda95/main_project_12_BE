# 🍽️ AI 프런트엔드 & 응답품질 템플릿 매뉴얼 v1.1 (수정·통합)

> 목적: “음식·영양 정보 특화” 서비스에 맞춘 **AI 응답(Self-Check)**, **추천질문(stats)**, **프런트엔드 규격**을 표준화한다.  
> 백엔드 명세(API Spec)를 기준으로 하며, 프론트 전용 규칙만 보완한다.  
> 각 항목의 `☑︎ 체크리스트`와 **수용 기준(AC)** 포함.

## 5) 네트워크 규칙
- 요청 키: (method,url,body) → 중복요청 취소  
- 타임아웃: 6s, 408 UX 제공  
- 자동 재시도 금지 (사용자 주도만)  
- 에러 매핑:  
  - 401/403/404/422/429 → 원인별 메시지  
  - 5xx → 사과/재시도 안내  
- 지표 로깅: latency_ms, hits, status  

☑︎ 체크리스트  
- [ ] AbortController 적용  
- [ ] 6s 타임아웃 중앙화  
- [ ] 408/429/5xx UX 구현  
- [ ] latency_ms/hits 로깅  

AC: p95 응답 ≤ 900ms, 타임아웃률 ≤ 1%/일

## 6) DTO / 타입

### FoodAnswer
```ts
export interface FoodAnswer {
  nutrition: string;
  allergy: string;
  storage: string;
  processing: string;
  source: string;
  meta?: {
    check_pass: boolean;
    retry_used: boolean;
    violations?: { code: string; detail?: string }[];
    latency_ms?: number;
    hits?: number;
  };
}
```

- `/inference` 응답(JSON)을 매핑해 사용한다.  
- `"정보 부족"`은 숨기지 않고 그대로 노출.  

### 추천 관련 타입
```ts
export interface RecSuggestion { text: string; reason: "co-occur" | "popular" | "trend"; }
export interface RecPrefixResponse { suggestions: RecSuggestion[]; source: "stats"; }
export interface RecPopularResponse { items: { text: string; score: number; window: "day"|"week" }[]; source: "stats"; }
```

⚠️ 주의: 백엔드 기준으로 `/api/v1/queries/recommended`, `/api/v1/search-logs/popular` 사용.  
트렌드(trend)는 프론트에서 인기 검색어를 가공하여 구현한다.

## 7) Self-Check (프론트 반영)
- 검사 항목: 4섹션 존재/순서, 단위 혼재, kcal↔탄단지 상충, 출처 최소 1개  
- 동작: 1차 응답 검사 → 실패 시 1회 보정 → 최종 실패 시 `"정보 부족"` 기록  
- 로그: meta.check_pass/retry_used/violations 반영  

AC: 형식 오류 p95 ≤ 1%/월, 출처 누락 p95 ≤ 2%/월

---

## 8) 추천/검색 연동
- `GET /api/v1/queries/suggest?prefix=...` → 자동완성  
- `GET /api/v1/queries/recent` → 최근 질의  
- `GET /api/v1/queries/recommended` → 추천 질문 (MV 기반, 수동 갱신)  
- `GET /api/v1/search-logs/popular?window=day|week` → 인기 검색어  

캐시:  
- 자동완성: 60–120s  
- 인기검색어: 5–10분  
- 추천질문: 10–30분

## 9) UI/UX
- **4탭 고정**: 영양 → 알레르기 → 보관 → 가공법  
- **출처(source)**: 우상단 패널  
- 상태 뷰: `<LoadingSkeleton/>`, `<EmptyState/>`, `<ErrorState/>`, `<NoAccess/>`  
- 탭 전환: 200–250ms 애니메이션  

AC: LCP ≤ 2.5s, INP p75 ≤ 200ms
