# 🍽️ AI 프런트엔드 & 응답품질 템플릿 매뉴얼 v1.0

> 목적: “음식·영양 정보 특화” 서비스에 맞춘 **AI 응답품질(Self-Check)**과 **추천질문(stats)**, **프런트엔드 규격**을 표준화한다.  
> 이 문서는 **템플릿**이며, 각 항목의 `☑︎ 체크리스트`와 **수용 기준(AC)**를 모두 포함한다.

---

## 0) 메타데이터
- 문서 버전: v1.0
- 적용 범위: Web Frontend(React SPA) + 품질검증(Self-Check) + 추천질문(stats) 연동
- 산출물: 정적 빌드 아티팩트(CDN/Nginx), Admin 대시보드(통계/품질)

---

## 1) 원칙(Principles)
1. **CSR(SPA)**: SSR 금지, 정적 산출물만 서빙.
2. **일관 네트워크 UX**: 중복요청 취소, 6s 타임아웃, 408/429/5xx 차등 처리, 자동 재시도 금지.
3. **A11y 준수**: WCAG AA, 포커스/트랩, 키보드 100% 내비게이션.
4. **상태 뷰 표준화**: 로딩/빈/에러/권한부족 뷰 공통 컴포넌트.
5. **프라이버시 우선**: 질의 원문 미저장(정규화/해시만), 최소수집.

5) 네트워크 레이어 표준
5.1 규칙

중복요청 취소: (method, url, body-key) 로 키 생성 → 직전 동일키 요청 AbortController 취소

타임아웃: 기본 6s, 타임아웃 시 408 UX(“다시 시도” CTA)

자동 재시도 금지: 사용자 주도 재시도만

에러 매핑

401/403/404/422/429: 원인별 메시지 + 다음 액션

5xx: 사과/재시도 + 상태페이지(선택)

지표 로깅: latency_ms, hits, status

5.2 API 에러 타입(예시)
```ts
export type ApiError =
  | { type: "TIMEOUT"; message: string }
  | { type: "HTTP"; status: number; message: string }
  | { type: "ABORTED"; message: string }
  | { type: "UNKNOWN"; message: string };
```

7) Self-Check(경량 품질 검증)
7.1 체크 항목
형식: 4섹션 존재/순서/빈값, 단위 혼재(100g vs 1회) 검사

상충: kcal ↔ 탄/단/지 합계 역설

출처: 섹션별 최소 1개 출처 문자열

7.2 동작
1차 응답 수신 → 규칙 검사

실패 시 보정 프롬프트 1회

최종 실패: 섹션 유지 + 빈값 "정보 부족", violations 기록

7.3 Self-Check 보정 프롬프트(예시)
text
코드 복사
당신의 이전 응답에서 다음 규칙을 위반했습니다: {violations}.
요구 형식: [영양, 알레르기, 보관, 가공법] 4섹션, 각 섹션 최소 1개 출처.
단위는 100g 기준(가능 시 1회분 병기). 누락/모호/상충 부분을 보완해 같은 형식으로 단 1회 재작성하세요.

8) SearchLog 기반 추천(stats)
8.1 로그 스키마(백엔드 제공, 프런트 의존)
query_norm(정규화), qhash(sha-256 일부), created_at

hits, latency_ms, top_categories(array)

PII 금지: 원문 질의 미보관, user_id는 NULL 또는 내부 ID

8.2 집계 파이프라인
주기: 5분 미니배치 + 일/주간 롤업

인기: count(qhash) Top-N

트렌드: 1h 대비 24h 가중 상승률(EWM)

연관: 코오커런스 기반(PMI 유사), 2~3gram

정화: 금칙어 필터, 편집거리(≈0.8)로 유사 질의 병합

8.3 API 계약
http
코드 복사
GET /api/v1/recommendations?q={prefix}
200 { "suggestions": [ { "text": "...", "reason": "co-occur|popular|trend" } ], "source":"stats" }

GET /api/v1/recommendations/trending?window=day|week
200 { "items": [ { "text":"...", "score":0.87, "window":"day" } ], "source":"stats" }
8.4 캐시 정책
prefix: 60–120s, trending: 5–10분
