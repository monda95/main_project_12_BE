# 사용자 요구설명서

> **1개월** 개발 + EC2 **t2.micro(프리티어)** 제약을 전제로 한 **음식·영양 정보 조회 중심의 AI 서칭 서비스**

## 1. 개요 및 목표
- **목표**: 음식·재료의 영양성분/알레르기/보관/가공법을 신속·명확히 제공하는 **AI 서칭 서비스(MVP)** 구축
- **기간/환경**: 1개월 개발, EC2 **t2.micro(프리티어)** 제약
- **특징**:
  - 비로그인 사용자도 AI 질의 가능 → 기록은 익명 저장  
  - 인증/로그인 시점부터 `role=user` 부여, 개인화/히스토리/통계에 반영  
  - 외부 LLM(Gemini) 결과에 **Self-Check** 단계 적용 → 실패 시 재검토 방식
  - "단일 질의 → 4섹션 응답(영양/알레르기/보관/가공법) + 출처"
  - Search API는 Gemini를 직접 호출하지 않고 InferenceService를 경유 → **코드 중복 제거 및 경량화**
  - 미래 계획: 대화 데이터 전처리 → `음식:영양성분` DB 구축

- **개발 범위 (MVP)**:
  - **포함**: 4섹션 응답, 인기/최근(사전집계), 자동완성(prefix), 관리자 지표(사전집계), 로그/헬스체크  
  - **제외**: 벡터DB·개인화 추천·SSR·실시간 대시보드·무거운 워커  
  - **프런트**: CSR(SPA) 빌드 산출물만 서빙, 요청 중복 취소(AbortController), 타임아웃, 429/408/5xx 에러 UX 표준화

| DH_RF04002 | 조회 통계 | 기능 | 가입자 수, 활성 유저 수, 인기 검색어(MV) | 중 | 관리자 전용 API, **stats 앱 제공** |
| DH_RF05001 | 추천 질문 | 선택 | 최근 검색어 기반 Gemini 실시간 호출 (캐시 없음) | 선택 | 추후 배치/캐시화 검토 |

---

### Notes
- **DH_RF03004 (Self-Check 규칙 검사)**  
  - 형식: nutrition / allergy / storage / processing / source 존재 여부  
  - 상충: kcal ↔ 탄·단·지 합계 역설 탐지  
  - 출처: 비어있지 않아야 함

- **DH_RF03005 (Self-Check 보정 재시도)**  
  - 1회만 재프롬프트  
  - 실패 시 `"정보 부족"`으로 저장  

- **DH_RF04002 (조회 통계)**  
  - 인기 검색어는 `PopularQuery` 모델을 통해 **MV(Materialized View)** 기반 관리  
  - 사전 집계된 데이터만 조회, 실시간 GROUP BY 금지  

- **DH_RF05001 (추천 질문)**
  - 최근 검색어를 읽어 Gemini API를 실시간 호출
  - 별도 MV나 캐시 없이 즉시 응답
  - 추후 t2.micro 리소스 여유 시 RecommendedQuestion 테이블 기반 캐싱/배치화 검토

## 3. 비기능 요구사항(NFR)
- **성능**: p50 ≤ 800ms, p95 ≤ 2000ms(캐시 적중), LLM 포함 p95 ≤ 3500ms
- **오류율**: 전체 ≤ 1%
- **가용성**: 단일 인스턴스(t2.micro), 장애 시 수동 재기동
- **보안/배포**: 운영배포 시 `DEBUG=False`, `ALLOWED_HOSTS` 지정, `SECURE_*`·`CSRF_*` 활성화

---

## 4. 운영 규칙
- **스로틀링**: 익명 20/min, 사용자 60/min (AI 추론/검색 API 중심)  
- **페이징**: 기본 page_size=20  
- **캐시**: 검색·통계 응답 TTL 30–120s  
- **LLM 호출 가드**: timeout 6s, 토큰 제한, 재시도=0  
- **집계**: 실시간 GROUP BY 금지, 사전 집계만 허용  

### Self-Check 규칙
- 재시도 1회, 실패 시 `"정보 부족"`으로 명시  
- 결과는 InferenceRun에 기록, 관리자 통계에서 확인 가능  
- 칼로리 상충·출처 확인은 O(1) 연산 → t2.micro 환경에서도 안전
---

## 5. 데이터 모델 특징
- **User**: role(guest/user/admin), `email_verified_at` 포함  
- **비로그인 허용**: Conversation, SearchLog에 user_id/owner_id=NULL 가능  
- **AI 응답**: Self-Check 결과와 4섹션 응답 스키마 저장  

### InferenceRun (추론 로그)
| 컬럼       | 타입    | 설명 |
|------------|--------|------|
| check_pass | BOOLEAN | Self-Check 통과 여부 |
| retry_used | BOOLEAN | Self-Check 재시도 여부 |
| violations | JSONB   | 규칙 위반 배열 |
