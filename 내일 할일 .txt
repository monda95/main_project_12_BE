#  ✅ 로컬 개발 단계 (Celery까지 다 켜야 하는가?)
#
#  기본적으로는 Django 서버(runserver/gunicorn)만 실행
#
#  단순 기능 개발/테스트 시 Celery worker/beat까지 켤 필요는 없습니다.
#
#  예: 검색/인증/CRUD API 개발 중이라면 runserver만으로 충분.
#
#  Celery 동작이 필요한 경우만 별도로 실행
#
#  예: "인기검색어 집계", "추천검색어 업데이트" 같은 주기적 작업을 테스트할 때.
#
#  이럴 때는 로컬에서 아래처럼 수동 실행:
#
#  uv run celery -A config worker -l info
#  uv run celery -A config beat -l info
#
#
#  즉, run.sh에서 주석 해둔 부분을 풀지 않고, 필요할 때만 터미널에서 개별 실행.

✅ 수동 실행 방법
1. Django Shell에서 실행
uv run python manage.py shell

from core.tasks import refresh_popular_queries_mv
refresh_popular_queries_mv.delay()   # 비동기 실행
refresh_popular_queries_mv()         # 즉시 실행(동기)

2. Celery worker/beat 없이 단발 실행
uv run python manage.py shell -c "from core.tasks import refresh_popular_queries_mv; refresh_popular_queries_mv()"

3. Celery 전용 명령어 (worker가 켜져 있을 때)
uv run celery -A config call core.tasks.refresh_popular_queries_mv
